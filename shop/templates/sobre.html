Escolhas/Alternativas

Usuário

Para registrar um usuário no meu site é necessário uma verificação no E-mail, para isso eu apenas fiz uma view para o preenchimento dos dados e outra view para a checkagem da token a verificação vai ser enviada no E-mail e o código é gerado com urlsafe_base64_encode(force_bytes(user.pk)). O usuário deve preencher os campos(E-mail, nome completo, Senha, confirme a senha), ultilizei o AbstractBaseUser e o AbstractBaseManagerUser do django.contrib.auth.models para fazer essas configurações, eu também personalizei o admin para esse novo modelo de usuário e criei forms para esse novo modelo

Carrinho

Eu preferir fazer o histórico do carrinho do usuário usando o django session, para que mesmo com os usuários deslogado consigam construir seu carrinho, devido a isso antes de usar a função logout(django.contrib.auth import logout) é preciso fazer um backup do carrinho dentro do session para que após o logout o carrinho seja restaurado. Eu tinha visto no stack-overflow como fazer uma custom LogoutView e era ultilizado a função logout, com testes eu percebi que o carrinho dentro do session era resetado, então eu customizei para ter um backup usando apenas uma variável dentro do python que após o logout o carrinho é recuperado, como eu sei que o session tem formato de dicionario eu poderia deletar as outras chaves(['_auth_user_id'], ['_auth_user_backend'], ['_auth_user_hash']), mas preferir ir pelo safe que é usar função do próprio django para depois recuperar através do backup, assim eu achei mais seguro. Caso o usuário faça login o carrinho do usuário estará dentro do mysql, e caso antes do login já tinha itens adicionados no carrinho dentro do banco de dados(mysql) esses novos itens apenas vão ser adicionados, então um usuário pode montar o carrinho no celular e acessar esse mesmo carrinho após o login em outros dispositivo
